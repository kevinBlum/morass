# Migrating Morass 0.4–0.6 consumers to 0.7

The 0.7 baseline consolidates the 0.5 craft-material foundation and 0.6 `ShellLayout` with stronger accessibility relationships and reproducible public npmjs releases. It also makes `Tabs` semantics explicit. Treat this as a deliberate consumer migration because a pre-1 caret range such as `^0.4.1`, `^0.5.1`, or `^0.6.0` does not select 0.7.

## 1. Install the selected baseline from npmjs

Remove any project-level `@effigy-analytics` mapping to GitHub Packages, then pin the migration:

```bash
npm install --save-exact @effigy-analytics/morass@0.7.0
```

npm registry selection is scope-wide. If the project still consumes private GitHub-only packages under `@effigy-analytics`, plan a registry/lockfile transition or move those packages to a separate scope instead of blindly replacing the mapping.

Keep one application-entry stylesheet import:

```ts
import "@effigy-analytics/morass/styles.css";
```

## 2. Choose the correct Tabs semantics

For a real tab interface, set `mode="tabs"` and pass the active panel as children. Morass then owns the tab IDs, `aria-controls`, tabpanel ID, and `aria-labelledby` relationship:

```tsx
<Tabs
  aria-label="Settings"
  mode="tabs"
  tabs={tabs}
  value={tab}
  onValueChange={setTab}
>
  {tab === "profile" ? <ProfileSettings /> : <SecuritySettings />}
</Tabs>
```

For filters or segmented choices that update a separate region, set `mode="selection"` and omit children. Morass exposes the buttons as an `aria-pressed` selection group instead of claiming a tabpanel relationship. Both modes are explicit so conditional children cannot silently change semantics at runtime. If a tab-mode `value` matches no item, Morass keeps the first tab selected and reachable and labels the supplied panel from it; keep controlled values synchronized to avoid that recovery state.

## 3. Recheck forms and dialogs

No source change is normally needed:

- `TextField` and `SelectField` now generate stable IDs, associate help and error text, preserve caller `aria-describedby`, and mark rendered errors invalid.
- `Modal` now derives its accessible name from the visible title and uses a guaranteed `:focus` outline on its initially focused panel.

Remove tests or workarounds that expected help/error copy to become part of a field's accessible name. Continue providing accessible names for icon-only actions and application-specific validation announcements.

## 4. Reconcile themes and private overrides

The 0.5 material foundation changed the default canvas and surface and introduced felt/material tokens. JavaScript theme records passed to `validateTheme` should spread the current base before overriding:

```ts
import { themes } from "@effigy-analytics/morass";

const productTheme = {
  ...themes.light,
  "--m-color-primary": "#115c56",
};
```

If preserving the older 0.4 canvas is important during migration, override only its base colors and let new tokens inherit 0.7 defaults:

```css
:root {
  --m-color-bg: #f7f4ef;
  --m-color-surface: #ffffff;
}
```

Replace selectors targeting internal component classes such as `.m-card__*`, `.m-button__*`, and `.m-app-frame__*` with documented tokens, public material utilities, supported props, or a product-owned wrapper. Internal DOM structure is not a compatibility surface.

## 5. Adopt additive primitives where useful

- `PageHeader`, `EmptyState`, and `NotFound` arrived in 0.5.
- `Card.subtitle` and `Card.noPadding` arrived in 0.5.1.
- `ShellLayout` arrived in 0.6 and is optional; applications keep ownership of router links and product copy.

Do not convert domain-specific controls merely to maximize Morass usage. Shared generic behavior belongs in Morass; product semantics stay in the product.

## 6. Verify the consumer

Run the repository's tests, typecheck, and production build, then check:

- primary, secondary, danger, field, status, progress, tab/selection, and Modal states;
- explicit light, explicit dark, and OS-selected themes;
- keyboard-only focus and Modal restoration;
- forced colors and reduced motion;
- mobile layout and 200% zoom/reflow;
- `validateTheme` against the final computed token values;
- no unauthenticated install is redirected to GitHub Packages.

For the Effigy hosted showcase, also confirm `/morass` displays the dependency version actually installed in that build. The showcase migration is tracked separately from the package release so an unreleased branch never gates the live site.
